## Story

**As a** recipient do tracking stack,  
**I want** a coluna `consent_at` em `event_log`,  
**so that** tenha trilha de auditoria LGPD registrando o timestamp do consentimento para cada evento capturado.

## Acceptance Criteria

1. Nova migration criada em `migrations/` com número sequencial após o último existente (verificar qual é)
2. Migration adiciona coluna `consent_at INTEGER` (UNIX timestamp, nullable) em `event_log`
3. `functions/tracker.js` atualizado para aceitar campo `consent_at` no body do POST e persisti-lo
4. `docs/schema.md` atualizado com a nova coluna documentada
5. Migration é idempotente (usa `ALTER TABLE ... ADD COLUMN IF NOT EXISTS` ou verifica existência)

## Dev Notes

**Convenção de migrations neste projeto:** arquivos em `migrations/` são numerados sequencialmente (0001, 0002, ..., 0015 — 0005 foi intencionalmente pulado). Verificar o último antes de criar.

**Padrão do tracker.js:** todo campo do body é lido via desestruturação e passado para D1 com `.bind()`. Nunca interpolação de string — regra hard do projeto.

**Coluna nullable:** `consent_at` deve ser `INTEGER` nullable para não quebrar eventos históricos e eventos vindos de contextos sem CMP.

**Arquivos afetados:**
- `migrations/0016_add_consent_at_to_event_log.sql` (novo)
- `functions/tracker.js`
- `docs/schema.md`
