## Story

**As a** recipient do tracking stack que usa Kiwify,  
**I want** o adapter Kiwify verificando a assinatura HMAC-SHA1 do body,  
**so that** webhooks forjados sejam rejeitados antes de processar qualquer compra.

## Acceptance Criteria

1. `functions/webhook/kiwify/[slug].js` atualizado para verificar assinatura HMAC-SHA1 do body da request
2. A assinatura é lida do header `X-Kiwify-Signature` (verificar documentação Kiwify — pode ser query param)
3. Verificação usa `timingSafeEqual` de `functions/webhook/_utils.js`
4. Request com assinatura inválida retorna `401 Unauthorized`
5. `KIWIFY_HMAC_SECRET` adicionado em `wrangler.toml.example` como env var
6. `docs/platforms/kiwify.md` atualizado com instruções de onde obter o HMAC secret no painel Kiwify

## Dev Notes

**Cloudflare Workers Crypto:** usar `crypto.subtle.importKey` e `crypto.subtle.sign` para HMAC-SHA1. Não há `crypto.createHmac` (Node.js) no edge runtime.

**Leitura do body:** o body precisa ser lido para verificação HMAC antes de fazer `json()`. Ler como `text()` → calcular HMAC → `JSON.parse()`.

**`timingSafeEqual` já existe em `functions/webhook/_utils.js`** — verificar se aceita `Uint8Array` ou string.

**Formato HMAC Kiwify:** verificar documentação para confirmar se é SHA1 ou SHA256 e o nome exato do header/param da assinatura.
