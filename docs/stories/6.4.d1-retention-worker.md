# Story 6.4: D1 Retention Policy Worker

**Status:** To Do  
**Prioridade:** P2  
**Executor:** @dev  

## Objetivo
Implementar um Cloudflare Worker Cron Trigger para automatizar a limpeza de logs antigos, garantindo que o banco de dados D1 não exceda limites de armazenamento e mantenha performance.

## Requisitos Técnicos
1. **Worker Cron:** Criar script de limpeza em `functions/scripts/retention-policy.js`.
2. **Configuração:** Permitir configurar o número de dias de retenção via variável de ambiente `RETENTION_DAYS`.
3. **Wrangler:** Configurar o trigger no `wrangler.toml` (ex: semanalmente).

## Definição de Pronto (DoD)
- [ ] Script deleta registros de `event_log` e `sync_log` anteriores ao período definido.
- [ ] Logs de execução da limpeza registrados.
- [ ] Configuração de trigger presente no `wrangler.toml.example`.
