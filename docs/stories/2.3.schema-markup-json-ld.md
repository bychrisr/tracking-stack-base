# Story 2.3: Adicionar Schema markup JSON-LD nos examples

**Status:** Draft  
**Epic:** 02 — SEO / AEO / Visibilidade IA  
**Sprint:** 1  
**executor:** @dev  
**quality_gate:** @qa  

## Story

**As a** recipient do tracking stack,  
**I want** Schema markup JSON-LD já incluído nos templates de exemplo,  
**so that** meus produtos apareçam em rich results do Google e sejam melhor entendidos por IAs.

## Acceptance Criteria

1. Schema `Organization` adicionado em `examples/lead-form-page/index.html` antes do `</head>`
2. Schema `Organization` adicionado em `examples/sales-page/index.html` antes do `</head>`
3. Schema `FAQPage` incluído como bloco HTML **comentado** nos dois examples (para ativar quando houver FAQ)
4. Todos os valores são placeholders claramente marcados (ex: `"Nome do Negócio"`, `"https://seusite.com"`)
5. Comentário orienta o recipient a validar no Google Rich Results Test

## Tasks / Subtasks

- [ ] Ler `examples/lead-form-page/index.html` para identificar posição antes do `</head>` (AC: 1)
  - [ ] Inserir bloco `<script type="application/ld+json">` com Organization schema
- [ ] Mesmo procedimento em `examples/sales-page/index.html` (AC: 2)
- [ ] Adicionar FAQPage schema como bloco comentado nos dois arquivos (AC: 3)
- [ ] Verificar que todos os placeholders estão claramente marcados (AC: 4)
- [ ] Adicionar comentário com link do Google Rich Results Test (AC: 5)
